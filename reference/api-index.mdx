This page lists every public identifier in the library, organized by the header file that defines it. Include the header to get access to the identifiers it exports.

<AccordionGroup>

<Accordion title="async/allocator.hpp">

| Identifier | Description |
|---|---|
| `allocator` | Concept for a type that satisfies the allocator requirements |
| `allocator_of_t` | The allocator type returned by `get_allocator` |
| `get_allocator` | Query tag used to retrieve an allocator from a sender's attributes |

</Accordion>

| Identifier | Description |
|---|---|

| `set_value` | Completion tag used to complete on the value channel |

</Accordion>

<Accordion title="async/concepts.hpp">

| Identifier | Description |
|---|---|
| `multishot_sender<S>` | Concept modelled by senders where `connect` may operate on lvalues |
| `operation_state<O>` | Concept modelled by operation states |
| `receiver<R>` | Concept modelled by receivers |
| `receiver_base` | Empty base type; deriving from it opts in to modelling the `receiver` concept |
| `receiver_from<R, S>` | Concept modelled by a receiver `R` that handles what sender `S` sends |
| `scheduler<S>` | Concept modelled by schedulers |
| `sender<S>` | Concept modelled by senders |
| `sender_base` | Empty base type; deriving from it opts in to modelling the `sender` concept |
| `sender_in<S, E>` | Concept modelled by a sender `S` whose completion signatures depend on environment `E` |
| `sender_of<S, Sig, E>` | Concept modelled by a sender `S` that may complete with `Sig` given environment `E` |
| `sender_to<S, R>` | Inverse of `receiver_from<R, S>` |
| `singleshot_sender<S>` | Concept modelled by senders where `connect` operates on rvalues only |

</Accordion>
